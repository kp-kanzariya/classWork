Mouse Events

    click: Fired when a pointing device button (usually a mouse's left button) is pressed and released on an element.
    dblclick: Fired when a pointing device button is clicked twice on an element.
    mousedown: Fired when a pointing device button is pressed on an element.
    mouseup: Fired when a pointing device button is released over an element.
    mouseenter: Fired when a pointing device is moved onto the element that has the listener attached.
    mouseleave: Fired when a pointing device is moved off the element that has the listener attached.
    mousemove: Fired when a pointing device is moved while over an element.
    mouseover: Fired when a pointing device is moved onto the element that has the listener attached or onto one of its
    children.
    mouseout: Fired when a pointing device is moved off the element that has the listener attached or off one of its
    children.
    contextmenu: Fired when the right button of the mouse is clicked (before the context menu is displayed).

    Keyboard Events

    keydown: Fired when a key is pressed down.
    keyup: Fired when a key is released.
    keypress: Fired when a key is pressed down and released (deprecated).

    Form Events

    focus: Fired when an element has received focus.
    blur: Fired when an element has lost focus.
    change: Fired for input>, select>, and textarea> elements when the value changes.
    input: Fired when the value of an input>, select>, or textarea> element is changed.
    submit: Fired when a form is submitted.
    reset: Fired when a form is reset.

Document Events

    DOMContentLoaded: Fired when the initial HTML document has been completely loaded and parsed.
    load: Fired when a resource and its dependent resources have finished loading.
    unload: Fired when the document or a child resource is being unloaded.
    beforeunload: Fired when the window, the document and its resources are about to be unloaded.

Clipboard Events

    copy: Fired when the user initiates a copy action.
    cut: Fired when the user initiates a cut action.
    paste: Fired when the user initiates a paste action.

Drag and Drop Events

    drag: Fired when an element or text selection is being dragged.
    dragstart: Fired when the user starts dragging an element or text selection.
    dragend: Fired when a drag operation is being ended.
    dragenter: Fired when a dragged element or text selection enters a valid drop target.
    dragover: Fired when a dragged element or text selection is being dragged over a valid drop target.
    dragleave: Fired when a dragged element or text selection leaves a valid drop target.
    drop: Fired when an element or text selection is dropped on a valid drop target.

Touch Events (for mobile devices)

    touchstart: Fired when one or more fingers touch the screen.
    touchmove: Fired when one or more fingers move on the screen.
    touchend: Fired when one or more fingers are removed from the screen.
    touchcancel: Fired when a touch event is interrupted.

Focus Events

    focusin: Fired when an element is about to receive focus.
    focusout: Fired when an element is about to lose focus.

Pointer Events (for handling different input devices consistently)

    pointerdown: Fired when a pointer becomes active.
    pointerup: Fired when a pointer is no longer active.
    pointermove: Fired when a pointer changes coordinates.
    pointerenter: Fired when a pointer enters the bounding box of an element.
    pointerleave: Fired when a pointer leaves the bounding box of an element.
    pointerover: Fired when a pointer enters an element.
    pointerout: Fired when a pointer leaves an element.
    pointercancel: Fired when the browser determines that there is no longer any chance for the pointer to generate events.

Miscellaneous Events

    wheel: Fired when a wheel button of a pointing device (usually a mouse) is rotated.
    resize: Fired when the document view is resized.
    scroll: Fired when an element's scrollbar is being scrolled.
    error: Fired when an error occurs while loading an external file.
    animationstart: Fired when a CSS animation starts.
    animationend: Fired when a CSS animation ends.
    animationiteration: Fired when a CSS animation is repeated.
    transitionstart: Fired when a CSS transition starts.
    transitionend: Fired when a CSS transition completes.

    The Browser Object Model (BOM) in JavaScript refers to the interactions with the browser window and its components, such as the document, history, location, and navigator objects. BOM events are events triggered by actions related to these components. Here's a list of common BOM events:
    Window Events
    
    load: Fired when the whole page has loaded, including all dependent resources such as stylesheets and images.
    window.addEventListener('load', () => {
        console.log('Page fully loaded');
    });

    unload: Fired when the document or a child resource is being unloaded.
    window.addEventListener('unload', () => {
        console.log('Page is unloading');
    });
    
    beforeunload: Fired when the window, document, and its resources are about to be unloaded.
    window.addEventListener('beforeunload', (event) => {
        event.returnValue = 'Are you sure you want to leave?';
    });
    
    resize: Fired when the window is resized.
    window.addEventListener('resize', () => {
        console.log('Window resized');
    });
    
    scroll: Fired when the document view is scrolled.
    window.addEventListener('scroll', () => {
        console.log('Window scrolled');
    });
    
    error: Fired when a runtime script error occurs.
    window.addEventListener('error', (event) => {
        console.log('Error occurred: ', event.message);
    });
    
    focus: Fired when the window gains focus.
    window.addEventListener('focus', () => {
        console.log('Window focused');
    });
    
    blur: Fired when the window loses focus.
    window.addEventListener('blur', () => {
            console.log('Window lost focus');
        });
    
    *** Network Events ***
    
    online: Fired when the browser has gained network connectivity.
    window.addEventListener('online', () => {
        console.log('Network connection restored');
    });
    
    offline: Fired when the browser has lost network connectivity.
    window.addEventListener('offline', () => {
            console.log('Network connection lost');
        });
    
    *** History Events ***
    
    popstate: Fired when the active history entry changes while navigating the session history.
    window.addEventListener('popstate', (event) => {
            console.log('History changed: ', event.state);
        });
    
    *** Storage Events ***
    
    storage: Fired when a storage area (localStorage or sessionStorage) has been changed in the context of another document.
    window.addEventListener('storage', (event) => {
            console.log('Storage changed: ', event.key);
        });
    
    *** Fullscreen Events ***
    
    fullscreenchange: Fired when the document switches into or out of fullscreen mode.
    document.addEventListener('fullscreenchange', () => {
        console.log('Fullscreen mode changed');
    });
    
    fullscreenerror: Fired when there is an error in changing the document's fullscreen mode.
    document.addEventListener('fullscreenerror', () => {
            console.log('Fullscreen error occurred');
        });
    
    *** Clipboard Events ***
    
    copy: Fired when the copy operation is initiated.
    document.addEventListener('copy', () => {
        console.log('Copy operation initiated');
    });
    
    cut: Fired when the cut operation is initiated.
    document.addEventListener('cut', () => {
        console.log('Cut operation initiated');
    });
    
    paste: Fired when the paste operation is initiated.
    document.addEventListener('paste', () => {
            console.log('Paste operation initiated');
        });
    
    *** Orientation Events ***
    
    orientationchange: Fired when the orientation of the device changes.
    window.addEventListener('orientationchange', () => {
            console.log('Device orientation changed');
        });
    
    *** Battery Events ***
    
    levelchange: Fired when the battery level changes (part of the Battery Status API).
    navigator.getBattery().then(battery => {
        battery.addEventListener('levelchange', () => {
            console.log('Battery level changed');
        });
    });
    
    chargingchange: Fired when the charging state of the battery changes.
    navigator.getBattery().then(battery => {
        battery.addEventListener('chargingchange', () => {
            console.log('Battery charging state changed');
        });
    });
    
    chargingtimechange: Fired when the battery charging time changes.
    navigator.getBattery().then(battery => {
        battery.addEventListener('chargingtimechange', () => {
            console.log('Battery charging time changed');
        });
    });
    
    dischargingtimechange: Fired when the battery discharging time changes.
    navigator.getBattery().then(battery => {
            battery.addEventListener('dischargingtimechange', () => {
                console.log('Battery discharging time changed');
            });
        });
    
    These BOM events help manage and respond to changes in the browser environment, allowing developers to create more interactive and responsive web applications.
